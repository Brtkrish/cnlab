=====================================
STOP AND WAIT ARQ ALGORITHM
=====================================

SERVER (RECEIVER) ALGORITHM:
-----------------------------
1.  Start
2.  Declare necessary variables including server, newSock, expectedFrame,
    buffer, frame, data and an array for random values
3.  Create a socket for TCP communication using socket(PF_INET, SOCK_STREAM, 0)
    If failed → print error and exit
4.  Set up the server address structure (servAddr) with appropriate values
    for family, port and IP address
5.  Bind the socket to the server address using bind()
    If failed → print error and exit
6.  Listen for incoming connections with a backlog of 5 using listen()
    If failed → print error and exit
7.  Accept a connection from client using accept(), storing in newSock
    If failed → print error and exit
8.  Initialize expectedFrame = 0
9.  Loop continuously:
    9.1  Receive data from client into buffer using recv()
         If failed → print error and exit
    9.2  Extract frame number and message data from buffer
    9.3  Display the received frame number and its data
    9.4  Generate or read a random value to simulate ACK loss
    9.5  If the random value indicates ACK loss:
         → Print "ACK LOST"
         → Do not send ACK and wait for retransmission
    9.6  Else if received frame number equals expectedFrame:
         → Increment expectedFrame
         → Send ACK containing next expected frame number using send()
         → Display ACK sent message
    9.7  Else (duplicate frame received):
         → Resend previous ACK
         → Display duplicate frame message
10. Close client socket and server socket using close()
11. Stop


CLIENT (SENDER) ALGORITHM:
---------------------------
1.  Start
2.  Declare necessary variables including client, frame, buffer, data
    and an array for random values
3.  Create a socket for TCP communication using socket(PF_INET, SOCK_STREAM, 0)
    If failed → print error and exit
4.  Set up the server address structure (servAddr) with appropriate values
    for family, port and IP address
5.  Connect to the server using connect()
    If failed → print error and exit
6.  Initialize frame number = 0
7.  Loop continuously:
    7.1  Accept message input from the user
    7.2  Attach frame number with the message
    7.3  Send the frame to the server using send()
    7.4  Display frame sent message along with random value
    7.5  Wait for ACK from server using recv()
    7.6  If ACK is not received (timeout or loss):
         → Print timeout or ACK lost message
         → Retransmit the same frame
    7.7  Else if ACK received:
         → Display ACK received message
         → Increment frame number
8.  Close the socket using close()
9.  Stop


KEY RULE:
---------
Send 1 frame → Wait for ACK → Only send next frame after ACK received
If ACK is lost → retransmit the same frame

ROLES:
------
Server = RECEIVER
Client = SENDER
